Add `X-Priority: high` to skip pacing delay:

```sh
curl -H "X-Priority: high" \
  "http://localhost:8985/europe/riot/account/v1/accounts/by-riot-id/Someone/EUW1"
```

Without the header, requests use normal priority and are paced evenly across the remaining rate-limit window. **High-priority requests run as soon as a slot is available** — they skip the pacing spread but still respect Riot's app and method limits. This is a scheduling hint, not a rate-limit bypass.

RiftRelay only recognizes `high`. Any other value (or no header) means normal priority.

## When to use it

Default to normal priority. Reserve `high` for genuinely user-facing or latency-sensitive traffic. If everything is high priority, nothing is — you just lose the benefit of pacing.

If you have metrics enabled, keep an eye on queue depth and rejection rates. A spike in high-priority traffic can starve normal requests.

## Swagger UI

When Swagger is enabled, the OpenAPI spec includes `X-Priority` as a parameter so you can test priority behavior from the browser.
